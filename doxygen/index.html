Build the Asylo SDK

Prerequisites:

The Asylo toolchain must be installed to compile enclaves. See distrib/README.md for more details.

Furthermore, to use the SGX backend, you must have their prerequisites installed. The Asylo SDK uses Intel SGX SDK v1.9, so see their prerequisites https://github.com/intel/linux-sgx/blob/sgx_1.9/README.md "as of their v1.9 release"

The Asylo SDK builds with Bazel, Google's open source build tool. To install it in Ubuntu or Debian, run the following commands:

sudo apt-get install -y gnupg && { \
curl https://bazel.build/bazel-release.pub.gpg | sudo apt-key add -; } && { \
sudo cat > /etc/apt/sources.list.d/bazel.list <<EOF
deb [arch=amd64] http://storage.googleapis.com/bazel-apt stable jdk1.8
EOF
} ; sudo apt-get update && sudo apt-get install -y bazel

The SDK uses some extra tools too. Some code generation is done by Python: 

$ sudo apt-get install python

Enclave code

Targets for trusted enclave code and enclave test code must be built with a --config flag that specifies the enclave backend it will use.

For example, build the simple enclave smoke test with the SGX simulation backend with the following command: 

bazel build --config=sgx-sim //asylo/test/misc:enclave_smoke_test

For hardware-backed SGX, build with --config=sgx. Debug enclaves are signed with a key that does not support release builds. See TODO_MISSING_DOCUMENTATION for the SGX release workflow.

Untrusted code

Targets that are strictly for untrusted code do not need to be compiled with the --config=BACKEND flag. All targets in //asylo/platform/common may be used both inside and outside enclaves.
